# Hermes V2 — Custom LLM Training Data Strategy
> Complete pipeline for generating ultra-high-quality Chain-of-Thought
> training data to fine-tune your own UPSC-specialized LLM.

---

## 🎯 The Goal

Train a model that doesn't just output answers — it **thinks** like an
expert UPSC evaluator before answering. By feeding it the full internal
reasoning steps (Chain of Thought) generated by our DSPy + LangGraph
pipeline, your custom LLM will learn the **process** of answer generation,
not just the output.

**Target models:** Llama-3 8B/70B, Mistral 7B, Qwen-2.7B/7B/72B
**Training frameworks:** Unsloth, Axolotl, LLaMA-Factory, TRL

---

## 📊 Data Collection Architecture

Every time a user asks Hermes V2 a question, the LangGraph orchestrator
goes through 8 cognitive nodes. We record the **entire trajectory**.

### Chain of Thought (CoT) Capture

Each trajectory captures 7 structured reasoning steps:

```
Step 1 [topic_detection]:  "Analyzing question... Domain: Polity. Type: analytical.
                            Entities: Article 21, Maneka Gandhi. Weight: HIGH."

Step 2 [retrieval]:        "Retrieved 10 chunks using hybrid_and_graph strategy.
                            Top sources: qdrant, bm25, neo4j."

Step 3 [planning]:         "Selected framework: Thematic. Examiner persona: Constitutional
                            law expert. Trap: Confusing with judicial review.
                            Differentiator: Link to global constitutionalism."

Step 4 [drafting]:         "Drafted answer using Thematic framework. Word count: 850.
                            Incorporated 10 context chunks."

Step 5 [review]:           "Critique complete. Score: 0.95/1.0. Hallucinations: NONE."

Step 6 [revision]:         "Revision #1. Addressing critique feedback. Improved word count: 920."

Step 7 [verification]:     "Verification layers: fact_check=PASS, guardrails=PASS,
                            constitutional=PASS. Overall: ALL PASSED."
```

## 🏗️ Training Data Formats

### 1. ChatML / ShareGPT (SFT)
```json
{
  "messages": [
    {"role": "system", "content": "You are Hermes... <think> format..."},
    {"role": "user", "content": "Explain the Basic Structure Doctrine."},
    {"role": "assistant", "content": "<think>\nStep 1: Identify domain...\n</think>\n\nThe Basic Structure Doctrine..."}
  ],
  "metadata": {"quality_score": 0.95, "domain": "Polity"}
}
```

### 2. DPO Pairs (Direct Preference Optimization)
```json
{
  "prompt": "Explain the Basic Structure Doctrine.",
  "chosen": "The improved final answer (after revision)...",
  "rejected": "The initial draft (before critique)...",
  "metadata": {"critique_score": 0.95, "revisions": 1}
}
```

### 3. ORPO Pairs (Odds Ratio Preference Optimization)
```json
{
  "prompt": "Explain the Basic Structure Doctrine.",
  "chosen": "The improved final answer...",
  "rejected": "The initial draft...",
  "metadata": {"domain": "Polity"}
}
```

### 4. Reward Model Data
```json
{
  "answer": "The Basic Structure Doctrine...",
  "score": 0.95,
  "rubric_scores": {
    "critique_score": 0.95,
    "fact_check": 1.0,
    "guardrails": 1.0,
    "constitutional": 1.0
  }
}
```

### 5. Full Trajectory (Agent Learning)
```json
{
  "question": "Explain the Basic Structure Doctrine.",
  "cot_trace": [...],
  "final_answer": "...",
  "total_cost_usd": 0.003,
  "total_latency_ms": 8200,
  "models_used": ["gemini-2.5-flash", "gpt-oss-120b"]
}
```

## 🔒 Quality Gating (6 Constraints)

Only trajectories that pass **ALL** gates are saved to the training dataset:

| Gate | Threshold | Why |
|------|-----------|-----|
| Critique Score | ≥ 0.9 | Ensures expert-level quality |
| Fact Check | PASSED | Zero hallucinations |
| Guardrails | PASSED | No biased/hedging content |
| Revisions | ≤ 2 | Discourages inefficient looping |
| Answer Length | ≥ 200 words | Ensures substance |
| Constitutional Check | PASSED (if HIGH weight) | Verifies legal citations |

**Expected pass rate:** ~30-40% of all generated answers.
**This is intentional** — we only train on perfect data.

## 🚀 Usage: Fine-Tuning Your Custom LLM

### Step 1: Collect Data
```bash
# Run the system and let it collect data
docker compose up -d
# Use the API to generate answers
curl -X POST http://localhost/api/answer \
  -d "question=Explain the Basic Structure Doctrine&session_id=test1"
```

### Step 2: Export Training Data
```bash
# Manual export
curl -X POST "http://localhost/api/dataset/export?min_score=0.9&upload_to_minio=true"

# Or wait for the weekly automated export (Sunday 4AM)
```

### Step 3: Train with Unsloth
```python
from unsloth import FastLanguageModel
from datasets import load_dataset

# Load the ChatML data
dataset = load_dataset("json", data_files="dataset/exports/export_*/chatml_sft.jsonl")

# Fine-tune Llama-3 8B
model, tokenizer = FastLanguageModel.from_pretrained(
    "unsloth/llama-3-8b-bnb-4bit",
    max_seq_length=4096,
)

model = FastLanguageModel.get_peft_model(
    model,
    r=16,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
    lora_alpha=16,
    lora_dropout=0,
)

# Train
trainer = SFTTrainer(
    model=model,
    tokenizer=tokenizer,
    train_dataset=dataset["train"],
    dataset_text_field="text",
    max_seq_length=4096,
    args=TrainingArguments(
        per_device_train_batch_size=2,
        gradient_accumulation_steps=4,
        warmup_steps=5,
        max_steps=60,
        learning_rate=2e-4,
        fp16=not torch.cuda.is_bf16_supported(),
        bf16=torch.cuda.is_bf16_supported(),
        logging_steps=1,
        output_dir="outputs",
    ),
)
trainer.train()
```

### Step 4: Train with DPO
```python
from trl import DPOTrainer
from datasets import load_dataset

# Load DPO pairs
dataset = load_dataset("json", data_files="dataset/exports/export_*/dpo_pairs.jsonl")

# Train with preference optimization
dpo_trainer = DPOTrainer(
    model=model,
    ref_model=ref_model,
    args=DPOConfig(
        beta=0.1,
        learning_rate=5e-7,
        max_length=2048,
    ),
    train_dataset=dataset["train"],
)
dpo_trainer.train()
```

## 🔬 Advanced: GraphRAG Benchmark

The system supports benchmarking GraphRAG vs Hybrid retrieval:

```bash
# Run the benchmark
curl -X POST "http://localhost/api/benchmark/run?n_questions=500"

# Compare results
curl http://localhost/api/regression/v1-v2
```

If GraphRAG shows even a small improvement in answer quality (structural
scores, factuality, or deeper insights), it becomes the primary retrieval
mechanism.
